# Довідники

Довідники — це об'єкти-каталоги системи: номенклатура, контрагенти, склади, валюти тощо. Назва контролера вказана у полі `ocontroller` відповіді `definition`.

---

## Search

**POST** `/{controllerName}/search/`

Повертає список елементів довідника за заданими критеріями.

### Параметри запиту

```json
{
    "q": "пошуковий рядок",
    "ft": "full",
    "rf": 0,
    "f": 0,
    "df": 1,
    "mci": 100,
    "pa": null,
    "own": null,
    "gst": false
}
```

| Параметр | Тип | Опис |
|----------|-----|------|
| `q` | string | Пошуковий запит. Якщо не вказано — повертає всі доступні елементи (до ліміту `mci`) |
| `ft` | string | Рівень заповнення полів у відповіді: `"full"`, `"small"`, `"forlist"`, `"none"` |
| `rf` | int | Бітові прапорці пошуку (детально нижче) |
| `f` | int | Бітовий фільтр за `flags` — вказані біти повинні бути **присутні** |
| `df` | int | Бітовий фільтр за `flags` — вказані біти повинні бути **відсутні**. Типово `df: 1` щоб виключити видалені |
| `mci` | int | Max count items — максимальна кількість результатів. За замовчуванням визначається сервером |
| `pa` | GUID \| null | Parent — для ієрархічних довідників: обмежити пошук у межах батьківського елемента |
| `own` | GUID \| null | Owner — для об'єктів з власником (наприклад, штрихкоди номенклатури) |
| `gst` | bool | Get SubTree — `true` щоб рекурсивно шукати серед всіх нащадків `pa` |

### Параметр `ft` — рівень заповнення

| Значення | Опис |
|----------|------|
| `"full"` | Повна модель — всі поля заповнені |
| `"small"` | Скорочена модель — мінімальний набір полів (id, caption) |
| `"forlist"` | Рівень для відображення в списку — проміжний між `small` та `full` |
| `"none"` | Не повертати об'єкти в `data` |

### Параметр `rf` — бітові прапорці пошуку

| Біт | Опис |
|-----|------|
| `8` | Для ієрархічних довідників: шукати не лише на поточному рівні, але й у всіх вкладених |
| `16` | Для ієрархічних довідників: ігнорувати фільтр за батьком `pa` |
| `64` | Пошук за числовим id або службовим полем |
| `1048576` | Відключити розбивку запиту на слова (шукати рядок цілком) |
| `2097152` | Обов'язкове збігання довжини — точний збіг рядка |

**Точний пошук** (наприклад, за кодом чи артикулом):
```json
{ "q": "ABC-001", "rf": 3145728, "ft": "full" }
```
де `3145728 = 1048576 + 2097152`

### Приклади

```json
POST /nomenclature/search/

{ "q": "кава", "ft": "full" }
```

```json
POST /storage/search/

{ "ft": "full", "mci": 2000000 }
```

```json
POST /nomenclature/search/

{ "ft": "forlist", "pa": "guid-батьківської-групи", "gst": true, "mci": 200000000 }
```

### Формат відповіді

```json
{
    "data": [
        {
            "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
            "c": "Кава Арабіка 250г",
            "flags": 0
        }
    ],
    "message": null,
    "sw": 5
}
```

## List

**POST** `/{controllerName}/list/`

Повертає елементи довідника за переліком ідентифікаторів.

### Параметри запиту

```json
{
    "ft": "full",
    "ids": [
        { "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6" },
        { "id": "7c9e6679-7425-40de-944b-e07fc1f90ae7" }
    ]
}
```

| Параметр | Тип | Опис |
|----------|-----|------|
| `ft` | string | Рівень заповнення полів |
| `ids` | array | Масив об'єктів з полем `id` (internalGuid) |

### Формат відповіді

```json
{
    "data": [
        {
            "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
            "c": "Кава Арабіка 250г",
            "flags": 0,
            "parent": { "id": "...", "c": "Кава" }
        }
    ],
    "message": null,
    "sw": 3
}
```

Порядок елементів у `data` відповідає порядку в `ids`. Якщо об'єкт не знайдений — він відсутній у відповіді.

## Save

**POST** `/{controllerName}/save/`

Для довідників `save` виконує **повне збереження** — передається вся модель елемента.

### Поля-посилання на USAP-об'єкти

Поля, що посилаються на інші сутності системи, приймають **лише значення `id`** (internalGuid) цільового об'єкта у вигляді GUID-рядка — не повний об'єкт.

```json
"назва_поля": "3fa85f64-5717-4562-b3fc-2c963f66afa6"
```

### Створення нового елемента

Передати `id: null` або не передавати поле `id`:

```json
POST /kontragent/save/

{
    "data": {
        "id": null,
        "c": "ТОВ Нова Компанія",
        "flags": 0,
        "edrpou": "12345678"
    },
    "ft": "full"
}
```

### Оновлення існуючого елемента

```json
POST /kontragent/save/

{
    "data": {
        "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
        "c": "ТОВ Оновлена Компанія",
        "flags": 0,
        "edrpou": "12345678"
    },
    "ft": "full"
}
```

### Формат відповіді

```json
{
    "data": [
        {
            "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
            "c": "ТОВ Оновлена Компанія",
            "flags": 0
        }
    ],
    "message": null,
    "sw": 8
}
```

Якщо `message` не `null` — збереження не відбулося, значення містить причину.

## Definition

**POST** `/{controllerName}/definition/`

Повертає повний опис структури типу: поля, їхні типи, мітки, посилання на контролери. Використовується для розуміння моделі при інтеграції.

Детально описано у розділі [Definition](./5-Definition.md).
