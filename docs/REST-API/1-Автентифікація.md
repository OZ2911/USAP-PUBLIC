# Автентифікація

## Логін

Всі запити до API (крім `/login/`) вимагають активної сесії. Для отримання сесії необхідно виконати POST-запит на `/login/`.

**POST** `/login/`

```json
{
    "cid": "demo",
    "login": "admin",
    "password": "1",
    "u2fcode": "",
    "savecode": 0
}
```

| Поле | Тип | Опис |
|------|-----|------|
| `cid` | string | Ідентифікатор компанії (subdomain) |
| `login` | string | Логін користувача |
| `password` | string | Пароль |
| `u2fcode` | string | Код двофакторної авторизації (якщо увімкнено, інакше `""`) |
| `savecode` | int | `0` — не зберігати сесію, `1` — розширена тривалість сесії |

**Відповідь:**

```json
{
    "__sid": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
    "expair": "2026-04-14T23:59:59Z",
    "twofStatus": "OK",
    "mstoexpair": 86400000
}
```

Ключ сесії (`sk`) також встановлюється у **cookie** з тим же іменем `sk`.

---

## Передача ключа сесії

Для кожного наступного запиту ключ сесії передається **одним із двох способів**:

### 1. Cookie (рекомендовано для браузерів)

Браузер автоматично передає cookie `sk` після успішного логіну.

### 2. HTTP-заголовок (рекомендовано для серверних інтеграцій)

```
auth-key: <значення __sid з відповіді логіну>
```

Приклад:
```http
POST /nomenclature/search/ HTTP/1.1
Host: my.usap.online
auth-key: 3fa85f64-5717-4562-b3fc-2c963f66afa6
Content-Type: application/json

{ "q": "кава", "ft": "full" }
```

---

## Термін дії сесії та помилки

| HTTP-код | Причина |
|----------|---------|
| `200` | Успішно |
| `403` | Сесія відсутня або закінчилась. Потрібно повторно виконати `/login/` |
| `401` | Невірний логін або пароль |

Якщо відповідь прийшла з кодом `403`, необхідно повторно виконати логін та оновити ключ сесії.

---

## Приклад (curl)

```bash
# Отримати ключ сесії
SK=$(curl -s -c cookies.txt -X POST \
  -H "Content-Type: application/json" \
  -d '{"cid":"demo","login":"admin","password":"1","u2fcode":"","savecode":0}' \
  https://my.usap.online/login/ | jq -r '.__sid')

# Використати ключ у наступному запиті
curl -s -X POST \
  -H "auth-key: $SK" \
  -H "Content-Type: application/json" \
  -d '{"q":"кава","ft":"full"}' \
  https://my.usap.online/nomenclature/search/
```
