# Аналітичні звіти (analysisReport)

Звіти — це параметризовані табличні вибірки, що формуються збереженими процедурами SQL. Кожен звіт має унікальний ідентифікатор (`repId`), підтримує власний набір параметрів та повертає структуровану відповідь з даними, групуваннями та конфігурацією полів.

---

## Отримання переліку доступних звітів

**POST** `/analysisreport/search/`

```json
{ "df": 1 }
```

`df: 1` — виключає видалені. Відповідь — стандартний `JsonExt` з масивом об'єктів звітів.

Ключове поле кожного звіту — **`tsco`** — це і є `repId` для побудови URL запиту.

```json
{
    "data": [
        { "ic": "001", "c": "Продажі та каса за період", "tsco": "cash", ... },
        { "ic": "002", "c": "Реєстр переміщень", "tsco": "storageMovingRegister", ... }
    ]
}
```

---

## Виклик звіту

**POST** `/reports/analysis-{repId}`

Де `{repId}` — значення поля `tsco` зі списку звітів.

### Тіло запиту

```json
{
    "data": {
        "repid": "cash",
        "ds": "2026-01-01T00:00:00Z",
        "de": "2026-04-01T00:00:00Z"
    }
}
```

Поле `repid` обов'язкове — відповідає значенню `{repId}` в URL.

Набір інших параметрів (`ds`, `de`, `s`, `p` тощо) залежить від конкретного звіту та описаний у блоці `/*conf:*/` його SQL-процедури.

### Параметри залежно від типу

| Тип параметра | Формат значення |
|---------------|-----------------|
| Дата (`type: "date"`) | ISO рядок: `"2026-01-01T00:00:00Z"` |
| Об'єкт USAP (`type: "storage"`, `"pricetype"` тощо) | GUID рядок — `internalGuid` об'єкта без дефісів |

---

## Структура відповіді

### Зовнішній рівень

```json
{
    "data": "...",
    "billing": null,
    "currency_for_front": null,
    "sw": 142
}
```

| Поле | Опис |
|------|------|
| `data` | Результат звіту — може бути об'єктом або серіалізованим JSON-рядком |
| `billing` | Дані білінгу — надходять періодично по таймеру, у більшості відповідей `null` |
| `currency_for_front` | Довідник валют — надходить при першому запиті, а також після зміни курсів або редагування валюти; у решті відповідей `null` |
| `sw` | Час виконання, мс |

### Варіативність формату `data`

Поле `data` та вкладені поля (`data`, `dicdef`, `fields`, `groups`) можуть бути як готовими JSON-значеннями (об'єкт, масив), так і серіалізованими рядками — залежно від реалізації конкретного звіту. Перед зверненням до вкладених полів слід перевіряти тип значення і при потребі парсити рядок.

Крім того, результат може містити **один об'єкт** або **масив об'єктів**, якщо звіт повертає кілька датасетів одночасно.

Ці деталі можуть змінюватися з оновленнями звіту, якщо він не є кастомним для конкретної бази.

### Внутрішній рівень

```json
{
    "data": [...],
    "dicdef": {...},
    "fields": [...],
    "groups": [...]
}
```

---

## Рядки даних (`data`)

Масив рядків звіту. Кожен рядок містить значення ключів групувань та два об'єкти вимірів:

```json
[
    {
        "k": 3,
        "d": 15,
        "f": { "sum": 1250.00, "cnt": 7.000 },
        "fs": { "sum": 1250.00, "cnt": 7.000 }
    }
]
```

| Поле | Опис |
|------|------|
| Ключі групувань (`k`, `d`, ...) | Числові ідентифікатори елементів довідників — розшифровуються через `dicdef` |
| `f` | Значення вимірів для листового (деталізованого) рівня |
| `fs` | Значення вимірів для агрегованих рівнів (підсумки в групуваннях). Відсутнє, якщо повністю дублює `f` |

`f` та `fs` можуть відрізнятися, якщо звіт повертає, наприклад, суми у різних валютах на деталізованому рівні, але приведені до єдиної валюти — в агрегованому. Якщо значення ідентичні — `fs` може бути відсутнє, слід використовувати `f`.

---

## Довідники (`dicdef`)

Розшифровка значень ключів групувань та глобальні параметри звіту.

```json
{
    "values": {
        "caption": "Продажі та каса за період"
    },
    "k": {
        "controller": "kontragent",
        "items": [
            { "id": 3, "ig": "a1b2c3d4e5f6..." },
            { "id": 7, "ig": "b2c3d4e5f6a1..." }
        ]
    },
    "d": {
        "controller": null,
        "items": [
            { "id": 15, "c": "Готівка" },
            { "id": 16, "c": "Безготівка" }
        ]
    }
}
```

### `dicdef.values`

Глобальні параметри звіту.

| Поле | Опис |
|------|------|
| `caption` | Заголовок звіту |
| `pagebreak` | `true` — додати розрив сторінки після таблиці |
| `htmlInject` | HTML-код перед таблицею |
| `reportclass` | CSS-клас контейнера звіту |
| `groupswidth` | Ширина колонки групувань (у символах) |

### `dicdef[groupKey]`

Словник елементів для кожного ключа групування.

| Поле | Опис |
|------|------|
| `controller` | Назва контролера USAP (якщо елементи мають відповідність у системі), або `null` |
| `items` | Масив елементів довідника |

Структура елементів `items` залежить від типу довідника:

- **Сутність системи** (`controller` вказано): обов'язкові поля `id` (числовий внутрішній ключ) та `ig` (GUID). Поле `c` зазвичай **відсутнє** — назву слід отримувати через відповідний контролер за `ig`.
- **Самостійний довідник** (`controller: null`): обов'язкові поля `id` та `c` (назва). Поле `ig` відсутнє.

Значення ключів групування в рядках `data` відповідають `id` в `items` відповідного довідника.

---

## Конфігурація полів (`fields`)

Масив об'єктів, що описують виміри звіту. Порядок у масиві = порядок колонок.

```json
[
    {
        "caption": "Сума",
        "fieldKey": "sum",
        "enabled": true,
        "fractionLength": 2,
        "SignificantFractionLength": 0,
        "calculateTotalSum": true
    },
    {
        "caption": "Кількість",
        "fieldKey": "cnt",
        "enabled": true,
        "fractionLength": 3,
        "calculateTotalSum": false
    }
]
```

| Поле | Опис |
|------|------|
| `caption` | Заголовок колонки |
| `fieldKey` | Ключ у об'єкті `f` / `fs` рядка даних |
| `enabled` | `true` — відображати за замовчуванням, `false` — прихована, `null` — недоступна |
| `fractionLength` | Максимальна кількість знаків після коми (`-1` — не число) |
| `SignificantFractionLength` | Мінімальна кількість значущих знаків після коми |
| `calculateTotalSum` | Відображати підсумок у підвалі таблиці |
| `overwriteMaxWidth` | Максимальна ширина колонки в символах |
| `overwriteMinWidth` | Мінімальна ширина колонки в символах |
| `evalTemplate` | Функція обчислення значення (рядок-прототип) |
| `evalStyleTmpl` | Функція стилів клітинки (рядок-прототип) |
| `groupingKey` | Ключ для групування колонок (наприклад, за датою) |

---

## Конфігурація групувань (`groups`)

Масив об'єктів, що описують рівні групування. Порядок у масиві = вкладеність групувань.

```json
[
    {
        "groupKey": "k",
        "usapType": "kontragent",
        "caption": "Контрагент",
        "addAsDefault": true,
        "showChildren": true,
        "useHierarchy": false,
        "withElements": true
    },
    {
        "groupKey": "d",
        "usapType": null,
        "caption": "Тип оплати",
        "addAsDefault": true,
        "showChildren": false,
        "useHierarchy": false,
        "withElements": true
    }
]
```

| Поле | Опис |
|------|------|
| `groupKey` | Ключ групування — відповідає ключу в рядках `data` та ключу в `dicdef` |
| `usapType` | Тип сутності USAP у lowercase (`"storage"`, `"nomenclature"` тощо), `null` — немає відповідності |
| `caption` | Назва групування |
| `addAsDefault` | Групування активне за замовчуванням |
| `showChildren` | Елементи групування розгорнуті за замовчуванням |
| `useHierarchy` | Відображення у вигляді ієрархії |
| `withElements` | Відображати елементи довідника (при `false` — тільки ієрархія) |
| `sortBy` | Поле для сортування елементів (за замовчуванням — `c`) |

## Параметри звіту — блок `/*conf:*/`

Кожен звіт описує свої параметри у SQL-коментарі `/*conf:{...}*/`. Це конфігурація форми введення, не частина API-запиту безпосередньо — але саме її ключі є іменами параметрів у тілі запиту.

Приклад (звіт `storageMovingRegister`):

```json
{
    "params": {
        "ds": {
            "type": "date",
            "caption": { "uk": "Дата з" }
        },
        "de": {
            "type": "date",
            "caption": { "uk": "Дата до" }
        },
        "s": {
            "type": "storage",
            "caption": { "uk": "Склад" }
        },
        "p": {
            "type": "pricetype",
            "caption": { "uk": "Тип цін" }
        }
    }
}
```

Відповідний запит до API:

```json
{
    "data": {
        "repid": "storageMovingRegister",
        "ds": "2026-01-01T00:00:00Z",
        "de": "2026-04-01T00:00:00Z",
        "s": "bd71a951150d4cc9a800b9963302d08e",
        "p": "dda27befbf2d4d47a19c4d7adea26ecb"
    }
}
```

Параметри з `type: "date"` передаються як ISO рядок. Параметри з `type` що відповідає контролеру USAP — передаються як `internalGuid` об'єкта без дефісів. Необов'язкові параметри можна не передавати або передати `null`.

## Повний приклад — звіт `cash`

### Запит

```bash
curl -s -X POST \
  -H "auth-key: <ключ сесії>" \
  -H "Content-Type: application/json" \
  -d '{"data":{"repid":"cash","ds":"2026-01-01T00:00:00Z","de":"2026-04-01T00:00:00Z"}}' \
  "https://my.usap.online/reports/analysis-cash"
```

### Відповідь (зовнішній рівень)

```json
{
    "data": "{\"data\":[...],\"dicdef\":{...},\"fields\":[...],\"groups\":[...]}",
    "billing": null,
    "currency_for_front": null,
    "sw": 87
}
```

### Відповідь (після парсингу `data`)

```json
{
    "data": [
        {
            "ct": 2,
            "cu": 1,
            "cb": 4,
            "f": { "sum": 3500.00 },
            "fs": { "sum": 3500.00 }
        }
    ],
    "dicdef": {
        "values": { "caption": "Продажі та каса за період" },
        "ct": { "controller": null, "items": [{ "id": 2, "c": "Надходження" }] },
        "cu": { "controller": "currency", "items": [{ "id": 1, "c": "UAH", "ig": "..." }] },
        "cb": { "controller": "cashbox", "items": [{ "id": 4, "c": "Каса №1", "ig": "..." }] }
    },
    "fields": [
        { "caption": "Сума", "fieldKey": "sum", "fractionLength": 2, "calculateTotalSum": true }
    ],
    "groups": [
        { "groupKey": "cb", "usapType": "cashbox", "caption": "Каса", "addAsDefault": true },
        { "groupKey": "ct", "usapType": null, "caption": "Тип", "addAsDefault": true },
        { "groupKey": "cu", "usapType": "currency", "caption": "Валюта", "addAsDefault": false }
    ]
}
```
