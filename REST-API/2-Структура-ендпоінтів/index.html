    Структура ендпоінтів та формат відповіді

    URL-структура

    Всі ендпоінти бізнес-логіки мають уніфіковану структуру:

    POST /{controllerName}/{method}/

    Де controllerName — ім'я контролера відповідного типу об'єкта (в нижньому регістрі).

    Приклади:

    Об'єкт Controller name Метод URL
    Номенклатура nomenclature Search /nomenclature/search/
    Контрагент kontragent List /kontragent/list/
    Документ продажу storageshipment Save /storageshipment/save/
    Склад storage Definition /storage/definition/

    Перелік всіх доступних контролерів та їх назв можна отримати через /rootdocumentation/index.

    Доступні методи

    Метод URL Призначення
    search /{ctrl}/search/ Пошук / отримання списку об'єктів за критерієм
    list /{ctrl}/list/ Отримання повних моделей об'єктів за ідентифікаторами
    save /{ctrl}/save/ Створення або модифікація об'єкта
    job /{ctrl}/job/ Виконання команди над об'єктом або колекцією
    definition /{ctrl}/definition/ Опис структури типу для побудови UI
    index /{ctrl}/index/ Скорочений опис параметрів конфігурації типу
    print /{ctrl}/print/ Отримання моделі для друку

    Формат відповіді

    Всі відповіді API загорнуті в єдиний об'єкт JsonExt:

    {
    "data": [ /* масив об'єктів */ ],
    "message": null,
    "extmessage": null,
    "sw": 12
}
    Поле Тип Опис
    data array Масив результуючих об'єктів
    message string | null Текстове повідомлення (наприклад, помилка валідації бізнес-логіки)
    extmessage object | null Розширені дані від певних методів (наприклад, job)
    sw int Час виконання запиту на сервері (мілісекунди), для діагностики

    Якщо message не null — операція не виконалась, причина у значенні message.

    Формат посилальних полів (USAP-об'єкти)

    Поля, що посилаються на інші об'єкти системи, передаються та повертаються у вигляді:

    {
    "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
    "c": "Назва елемента"
}
    Поле Опис
    id internalGuid елемента (GUID)
    c Текстова назва/caption для відображення

    При збереженні об'єкта у посилальне поле достатньо передати { "id": "<GUID>" }, поле c необов'язкове.

    Content-Type

    Всі запити до API — POST з тілом у форматі JSON:

    Content-Type: application/json
